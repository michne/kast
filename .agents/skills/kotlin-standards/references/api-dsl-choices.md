# API DSL Choices

Router for API design questions. Prefer the narrowest file:

- `api-parameter-selection.md`: parameters, defaults, options, overloads.
- `api-builders-and-configuration.md`: builders, configuration objects, DSLs.
- `api-extensions-and-factories.md`: extensions, factories, parsers.
- `api-surface-stability.md`: visibility, compatibility, opt-in APIs.
- `api-review-guides.md`: final API review checklist.

Use a DSL only when structure or staged configuration beats plain named
arguments.
