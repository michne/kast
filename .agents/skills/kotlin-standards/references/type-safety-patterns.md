# Type Safety Pattern Router

Use the narrowest file:

- `types-domain-modeling.md`: value classes, sealed state, nullability removal,
  and immutability.
- `types-errors-and-testing.md`: typed outcomes, expected failures, and
  correctness tests.
- `types-dsls-and-generics.md`: DSL markers, receiver scopes, variance, and
  reified generics.

Default rule: choose the smallest type that prevents the invalid state you can
name.
